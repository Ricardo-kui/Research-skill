# Research Theory Builder Standard Input Template

Use this template when the user wants theory or hypothesis help but the request is underspecified.

## Minimum fields

- Project or paper title:
- Core phenomenon:
- Main outcome or dependent variable:
- Main explanatory variable:
- Candidate mechanism:
- Candidate moderator or boundary condition:
- Current theory section text:
  - none
  - outline
  - partial prose
  - full draft
- Current hypotheses, if any:
- Candidate theories already in play:
- Alternative explanation that worries the user:
- Research design or empirical setting:
- Measurement or proxy constraints:
- What the user wants now:
  - build mechanism
  - rewrite theory section
  - derive hypotheses
  - integrate multiple theories
  - test alternative logic

## Fast normalization rule

When the user does not provide these fields explicitly:

1. Infer the focal X and Y.
2. Identify the paper's likely main mechanism.
3. Separate constructs from measures and proxies.
4. Identify whether the real problem is mechanism, coherence, or hypothesis formulation.
5. State assumptions before rewriting if they materially affect the logic.

## Output modes

- `Mechanism-first`: Explain the logic chain before writing prose.
- `Hypothesis-first`: Draft hypotheses plus short lead-in logic.
- `Section rewrite`: Rewrite the full theory or hypothesis section.
- `Comparative`: Contrast competing theories and recommend one framing.
