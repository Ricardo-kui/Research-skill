# Hypothesis development 与 alternative logic playbook

## 一句话原则

好的假设不是把关系写出来，而是让读者承认：这个方向有理论理由，而且相反方向并非不可能。

## 假设构建的五步

1. 写清 `X -> Y` 的核心关系。
2. 解释为什么是这个方向。
3. 认真考虑反方向或零关系是否也有道理。
4. 说明在什么条件下该关系会增强、减弱或翻转。
5. 确认后续测量与设计真的能检验这个理论命题。

## 三类最常见假设

- `Direct effect`
  解释 X 为什么影响 Y。
- `Mediation`
  解释 X 通过什么过程影响 Y。
- `Moderation`
  解释 X 对 Y 的作用何时更强、何时更弱。

## 写假设前先 flirt with the null

一个很有用的动作是：

- 问自己为什么这个关系可能根本不存在。
- 问自己为什么相反方向也可能成立。
- 如果这两种可能性完全说不通，当前假设可能太 obvious。

## 假设要避免的四种坏写法

- `复述式`
  只是说 “X positively affects Y”，没有 why。
- `复合式`
  一个假设里同时塞多个关系。
- `测量式`
  在理论部分提前写成变量和代理的语言。
- `拼贴式`
  每个假设靠不同理论硬撑，整体没有统一主线。

## 调节假设尤其要注意

- 先讲清主效应，再讲调节。
- 解释的是 Z 如何改变 `X -> Y` 的机制，不是 Z 如何直接影响 Y。
- 若调节变量本身也影响 Y，必须把两套逻辑分开写。

## 常见误区

- 用前人显著结果代替自己的理论推导。
- 只因为文献常写 Hypothesis 1, 2, 3，就把逻辑切得很碎。
- 为了增加“完整性”堆很多假设，反而削弱主线。
- 讨论 moderation 时没有认真排除 mediation 或反向解释。
