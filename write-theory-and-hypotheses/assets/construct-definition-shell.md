# Construct Definition Shell

Use this scaffold when defining a new construct or clarifying an existing one that reviewers find ambiguous.

---

## Shell

```
[LABEL — state the construct name exactly as it will appear in the paper.]

[CORE DEFINITION — one to two sentences. State what the construct IS.
 Prefer: "We define [construct] as [the degree to which / the extent that / the process by which] ..."]

[SCOPE CONDITIONS — what counts as an instance, and what does not.
 At minimum: what level of analysis (individual, dyad, firm), what time horizon,
 what domain or context.]

[LINEAGE — which prior construct or theoretical tradition does this build on?
 If it is an extension: "Building on [prior construct] (Author, Year), we ..."
 If it is a new concept: "Unlike [neighboring construct], which focuses on X, [construct] captures Y."]

[DIFFERENTIATION — distinguish from the 1–2 closest neighboring constructs.
 Pattern: "[Construct A] captures [X]; our construct captures [Y], which differs
 because [Z]. Although related, the two are conceptually distinct because ..."]

[WHY THIS CONSTRUCT — one sentence justifying why this paper uses this construct
 rather than a near substitute. "We focus on [construct] rather than [alternative]
 because our theoretical argument centers on [mechanism], which [construct] more
 directly captures."]
```

---

## Differentiation Sentence Patterns

```
"Although [Construct A] and [our construct] both concern [shared domain],
they differ in [key dimension]: [Construct A] refers to [X], whereas [our construct] refers to [Y]."

"[Our construct] should not be confused with [Construct A].
[Construct A] captures [X at level/time/domain].
[Our construct] captures [Y at level/time/domain] and includes [distinguishing feature]."
```

---

## Red flags that a definition is weak

- [ ] Uses the construct name in the definition ("knowledge diversity is the diversity of knowledge…")
- [ ] No scope condition — applies to everything, therefore predicts nothing
- [ ] No differentiation from the nearest neighbor
- [ ] Appears after the first hypothesis — define before you theorize
- [ ] Definition does not match what is actually measured in Methods
