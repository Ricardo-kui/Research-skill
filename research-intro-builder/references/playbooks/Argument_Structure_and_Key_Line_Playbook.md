# Argument structure 与 key line playbook

## 一句话原则

好文章不靠读者“自己悟”，而靠作者不断给出主线、路标和推进顺序。

## 结构的最低要求

- 每一节只完成一种核心任务。
- 每一段只推进一个主点。
- 每一段的首句最好能说清本段在干什么。
- 各段之间要有显式连接，而不是靠读者自己脑补跳转。

## 金字塔结构怎么理解

- 最上层是全文主张：这篇文章最终要说服读者接受什么。
- 中间层是 section 级主点：引言、理论、方法、结果、讨论分别完成什么任务。
- 下层是段落级支撑点：每段用若干句子支撑该段的 key line。

如果一篇文章读起来像在“跳台阶”，通常不是内容不够，而是层级没分清。

## Key line 的三种高频用法

- `总起式`
  直接告诉读者这一部分接下来要证明什么。
- `连接式`
  把前文和后文扣在一起，说明为什么现在要转向下一步。
- `总结式`
  在一个局部结束时，把已经证明的点收束起来，防止主线松掉。

## 判断一段是否合格的快检

- 如果只看首句，读者能否知道本段任务？
- 如果只看尾句，读者能否知道这一段为何值得存在？
- 删掉中间细节后，首句和尾句是否仍然彼此呼应？
- 本段是否服务于上一级 section 的主任务，而不是顺手多说一些别的？

## 文献综述最该避免的问题

- 不要“总结别人的故事”，而要“讲自己的论证”。
- 当段落总是以作者名或年份开头时，通常已经接近文献罗列。
- 理论框架部分的任务不是展示你读了很多，而是把你的解释逻辑搭出来。

## 让文章更顺的几个动作

- 在 section 开头给出一句总起式 key line。
- 在 section 之间写出为什么要从 A 走到 B。
- 对核心主张进行必要重复，不要怕“说过一次了”。
- 动笔前先写提纲，确认主点、分点和顺序，而不是写到哪算哪。

## 常见误区

- 文献综述一段一个作者，彼此没有关系。
- 一段里同时做背景、理论、方法和贡献四件事。
- 只在脑中有主线，正文里却没有给读者任何路标。
- 重要观点只说一次，接下来就默认读者记住了。
- 为了显得 sophisticated，故意把关键句写得很隐晦。
